Что надо сделать в третьей лабе:
В лабе надо реализовать несколько методов поиска минимума. Отчетом является несколько графиков, по горизонтальной оси которых надо откладывать ограничение по количеству "расчетов" функции (обращений к таблице значений функции), по вертикальной оси - процент найденных верно ответов при 100 запусках функции. Пример: тестируется шаблонный метод. Пусть ограничение по расчету функции равно 50. Тогда надо 100 раз запустить этот метод, при этом при достижении 50 вычислений функции его работу закончить. Для каждого запуска надо проверить, совпадает ли найденная точка с фактической точкой минимума (которую, например, показывает метод полного перебора), и таким образом посчитать процент верно найденных решений при данном ограничении на расчеты функции.
Такую процедуру необходимо выполнить для всех алгоритмов, кроме перебора (там все стабильно), и для разных ограничений на количество расчетов функции например до 800 с шагом 50.
Также необходимо построить два разных графика: в одном проверять на точное попадание (скрин 1), а во втором правильным считается значение, которое попадает точно в точку минимума или в какую-либо соседнюю с ней (скрин 2).
К сообщению прикреплены примеры графика с входным файлом "Функция_П2". Сами графики не очень хорошими получились - можно подобрать параметры получше, а также можно учитывать что если к какому-то из значений функции уже было обращение, то в последующие разы обращение к тому же значению можно не считать новым расчётом функции и не увеличивать счётчик.
